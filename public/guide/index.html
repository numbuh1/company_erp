1 Getting Started

Logging In

Navigate to the application URL and sign in with your email and password. All features require authentication — you will be redirected to the login page if you are not signed in.

Access Control

What you can see and do depends on the permissions assigned to your role. If you try to access a page you do not have permission for, you will see a 403 Forbidden error.

ℹ️
Contact your administrator to request additional access. Permissions are managed through the Roles module.

4 Announcements

Company-wide posts with rich text formatting — bold, italic, lists, headings, and embedded images.

Viewing Announcements

The index page lists all announcements as cards. Click a title to read the full content on its own page.

Creating an Announcement

🔑
Requires the Create/Edit Announcements permission.
  1. Click New Announcement on the index page.
  2. Enter a Title.
  3. Write your content using the rich text editor. Use the toolbar for formatting.
  4. To insert an image, click the image icon in the toolbar — it uploads automatically.
  5. Click Save.

Editing & Deleting

  • Edit — available on the index card and the show page with Create/Edit Announcements permission.
  • Delete — requires Delete Announcements permission.

  • Assigned teams and members

7 Timesheet

Two views for tracking time — a filterable log list and a weekly grid.

Log List — /time-logs

A table of all time entries. Available filters: User, Team, Date, Project, Task.

Each row shows: Date · User (if visible) · Context (Task / Project / Other) · Description · Time Spent formatted as Xh Ym.

Logging Time

Click Log Time on the time-log index, project detail, or task detail pages.

FieldDescription
DateDate worked (defaults to today)
ProjectOptional — link to a project
TaskOptional — filtered to the selected project's tasks
DescriptionWhat you worked on
Time SpentDecimal hours (e.g. 1.5 = 1h 30m). Min: 0.25 · Max: 24
⚡
Quick-time buttons (30m, 1h, 2h, 4h, 8h) fill the time field instantly — no need to type.

Weekly View — /timesheets/weekly

A grid showing the current week. Days are columns; tasks/projects/other are rows.

  • Use Prev / Next to navigate weeks, or click This Week to return.
  • Hover over any time cell to see the log description in a tooltip.
  • The Today column is highlighted.
  • Day totals appear in the bottom row; the week total in the bottom-right.
  • Click a cell to view that log entry (or a filtered list if multiple entries exist that day).
ℹ️
With team/all timesheet permissions, use the User or Team filter to view other users' weekly logs.

8 Teams

Teams group users together. They are used for scoping leave/OT approvals, timesheet visibility, and project assignment.

Index Page

Lists all teams (or only your own teams, depending on your permission) with member count, leader names, and action buttons.

Team Detail Page

Shows two panels side by side:

  • Leaders — users who lead the team
  • Members — regular members

Creating / Editing a Team

🔑
Requires Create/Edit Teams.
  1. Click Create Team or the Edit button.
  2. Enter the Team Name.
  3. Check the users you want as members.
  4. Among the checked users, tick the Leader checkbox for those who should lead.
  5. Click Save.
💡
Leaders can approve leave and OT requests from their team members, and can view their team's timesheets.

9 Users

Manage user accounts, roles, and leave balances.

User Profile

Every user has a profile page showing:

  • Profile picture (or initial avatar), name, position, email
  • Roles assigned as badges
  • Teams — with Leader / Member label
  • Leave Balance in hours, with a link to the full change history (private info)
  • Leave Requests — a summary table of recent requests (private info)
ℹ️
User can see their own company private info. Other user without Edit All User Profile permission can't.

Editing Your Own Profile

Click Profile in the top-right profile menu to edit your own name, email, password, position, and profile picture.

ℹ️
Only users with Edit All User Profile can edit other user's company private info.

Managing Users (admin)

Leave Balance History

Click the Leave Balance History link on a user's profile to see a paginated log of all adjustments — change amount, balance after, reason, and who made the change.

10 Leave Requests

Request time off. Leave is measured in hours and deducted from your leave balance upon approval.

Submitting a Request

  1. Click New Request.
  2. Select the Type (e.g. Annual, Sick, Emergency).
  3. Set the Start and End date/time.
  4. Enter the Hours being consumed.
  5. Write a Description (reason for the leave).
  6. Click Submit.

Request Statuses

StatusMeaning
PendingAwaiting approval
ApprovedApproved — leave balance deducted
RejectedNot approved — a rejection reason is provided

Approving / Rejecting (approvers)

Users with Approve Team Leaves or Approve All Leaves see Approve and Reject buttons on pending requests.

  • Approve — sets status to Approved and notifies the requester.
  • Reject — prompts for a rejection reason, then notifies the requester.
🔔
The requester receives a bell notification the moment their request is approved or rejected.

11 Overtime (OT) Requests

Log overtime hours worked outside regular hours. The approval flow mirrors Leave Requests.

Submitting an OT Request

  1. Click New OT Request.
  2. Set the start time, end time, hours, type, and description.
  3. Click Submit.

Approval Flow

Approvers with Approve Team OT Requests or Approve All OT Requests can approve or reject. The requester receives a notification either way.

ℹ️
Approve Team OT Requests only covers requests from team members in teams you lead. You will not see requests from teams you belong to but do not lead.

12 Roles & Permissions

Roles bundle permissions together and are assigned to users to control what they can access.

🔑
Requires Roles module enabled to access this section.

Managing Roles

🔑
Requires Create/Edit Roles to create or edit.
  1. Go to Roles in the sidebar (Admin group).
  2. Click Create Role.
  3. Give the role a Name.
  4. Check the permissions to grant — grouped by module.
  5. Click Save.

Assigning Roles to Users

Roles are assigned from the User edit form — look for the Roles section when editing a user.

Deleting Roles

⚠️
Requires Delete Roles. Deleting a role removes it from all users currently assigned to it.

13 Notifications

Bell Dropdown

Click the 🔔 bell icon in the top bar to see your 5 most recent unread notifications. Each shows the sender's avatar, a title, a short description, and a relative timestamp. Click any notification to jump to the relevant page.

Click View All to open the full notifications page.

Notifications Page — /notifications

All notifications are listed here. Unread entries are highlighted with an indigo background. Opening this page marks all unread notifications as read.

When Are Notifications Sent?

14 Dark Mode

Click the ☀️ / 🌙 icon in the top navigation bar to toggle between light and dark themes.

Your preference is saved automatically and persists across sessions. On first visit, the theme follows your operating system's default.
